InfozenLab On-Board Guide

Below you can find three tabs: Tools, Code, and Run. These three tabs will be your Infozen 101. Here you can find all necessary guides and help on anything that would be needed at Infozen.

Install Tools

Java

1. Download and Install

1. Click here to go to the website.

2. Find Java SE Development Kit 8u45 on the page and click the bubble saying ‘Accept License Agreement’.

3. Then click the hyperlink next to Windows x64 under Java SE Development Kit 8u45. The file will start to download.

4. Once the file is finished downloading, double click it to start the install process. Click next for the first slide and on the second slide make sure your screen looks like this. If it does, click yes. If it doesn’t, change it to make it this.

5. After clicking next on that screen, java will start to install. Click next on the destination folder (just remember where the file is). Now java will finish installing.

6. Once java has finished installing, a screen that looks like this will pop up. Just click close to end the installation process.

Top of Page

2. Configure Java

1. Now you have to click the windows button in the bottom left corner of your screen and right-click on computer. From here, you will click properties.

2. Once you click properties, you will get a screen like this. From here, go to advanced system settings.

3. From the advanced system settings screen, click environment variables.

4. Now you have to click ‘new’ under ‘User Variables for (user)’. Once you do that for variable name, put ‘JAVA_HOME’. For the value, you will have to find the path to your java file.

5. To find the value, you will click your windows button in the bottom left-hand corner, and then click computer. Then click your hard disk drive, click ‘Program Files’ (make sure you click the one without x86). Next you will click the file called ‘jdk1.8.0_45’.

6. Here you will click the drop down arrow at the top of the page and then copy the path that is in the URL.

7. Then go back to the variable page. You will then paste the URL that you just coped into the variable value. Then click ok.

8. It should now show up under user variables.

9. Now find the system variable ‘Path’. Click it and then click edit. Once you are on the edit screen, make sure that in the variable value line, there is ‘%JAVA_HOME%\bin’. If there isn’t you will need to put it in after the first ‘\bin’. Then click ok.

Top of Page

3. Verify Java Version

1. You may now close all system windows for you are done with the installation. To make sure that the installation work, go to the command prompt. You can find this by clicking the windows button and putting command prompt in the search bar.

2. From the command prompt screen, you will type ‘java -version’. Make sure you put a space in between java and the ‘-‘symbol. Then hit enter. You should get a screen that looks like this.
>

3. Check to see what version of java you are running. The version number will be beside where it says ‘java version’. It should say ‘1.8.0_45’. If it doesn’t go back through the steps and make sure you did everything correctly. If it does, that means you have successfully finished installing java.

Top of Page

Git

1. Download and Install

1. Click here to go to the website.

2. Click the download for windows button on the right side of the page.

3. Once it has finished downloading, double click the file.

4. Click next to start the setup. Read through and then click next on the license.

5. For the components slide, don’t change anything. If you want to add a quick launch button for Git or add a desktop icon for Git, you can select that, but don’t change anything else. Then click next.

6. Make sure that the bubble highlighted is “Use Git from Git Bash only”. Then click next. On this slide, make sure the bubble highlighted is “Checkout Windows-style, commit Unix-style line endings”. Then click next. Git will start to install now.

7. Click finish and Git is now installed onto your computer.

2. Once you have opened the window, look at the top of the screen to find the version you have.

Spring Tool Suite

1. Download and Install

1. Click here to go to the website.

2. Once downloaded, right click the file and select 'extract all'. Make sure the folder is accessible and then click extract.

3. Once the file is open, click 'sts-bundle'. After that, you will click ‘sts-3.7.0.RELEASE'.

4. Then click STS (the application not the configuration settings) to start up the application.

Top of Page

2. Configure Server

1. Open up Spring Tool Suite. When asked for where you want your make sure you use 'C:\users\username'. This will come up later on when actually using the application.

2. Once the application is opened, find the server tab and right click on the screen. Once you right click, choose new, then click server.

3. Click 'Pivotal tc Server v3.0 - v3.1' and then click next.

4. Select creat new instance and then click next.

5. Name your server, then select the 'base' template. Then click finished.

6. Once it has been made, find it under the servers' tab. Right click the server, and then select start.

7. Next, go to your web browser and type in http://localhost:8080 to the url. After this, hit enter to see if creating the server worked. You should get a screen that looks like this:

Import Code From GitLab

Create SSH Key

1. Open up gitbash.

2. Type in 'ssh-keygen -t rsa -C "your email"'. Then hit enter. Just hit enter on the file loation and type in a passcode if you would like (not necessary).

3. Now go to the key location. It should be under 'C:\users\username\.ssh'. Once there, find the 'id_rsa.pub' file and open that with notepad.

4. Copy all the text whithin the file and then open up your browser. From you browser go to "iz-gitlab". Sign in and then go to profile settings. From there, click SSH keys.

5. Click the add SSH Key button, and then paste the text you copied earlier into the key box. If it doesn't automatically make a title, make one yourself and then click add key.

Ce-Tr-Broker

1. Open up Spring Tool Suite. Make sure your workspace is 'C:\users\username'.

2. Once in the application, right click under 'Package Explorer' and click import.

3. Once you click import, a screen will pop up asking from where you want to import. Click the 'Git' folder, and then click 'Projects from Git'.

4. Then click 'Clone URI'.

5. Next you will paste 'git@iz-gitlab:ccontrol/identrix-ce-tr-broker.git' into the URI and then click next.

6. You now will need to deselect feature and master, leaving dev the only one still marked. Then click next.

7. On the following screen, simply click next. After the application is done cloning, select import as general project and then click next.

8. Keep it's name how it is, then click finish. Once it pops up under the package explorer, right click it. From here, go to configure, then convert to mavin project.
